# AI-Driven SIEM with Real-Time Monitoring, UEBA, and SOAR

## Overview

This project is an AI-powered Security Information and Event Management (SIEM) system designed to detect, analyze, and respond to cybersecurity threats in real time. The system combines machine learning, User and Entity Behavior Analytics (UEBA), Apache Kafka streaming, and automated response capabilities (SOAR) to provide intelligent threat detection and security monitoring.

The platform continuously generates and processes security logs, analyzes user behavior, identifies anomalies, calculates risk scores, and automatically performs response actions such as IP blocking or rate limiting. All detected events and alerts are displayed through a real-time web dashboard.

---

## Key Features

### Real-Time Log Monitoring

* Uses Apache Kafka for real-time log streaming.
* Continuously processes security events as they occur.
* Eliminates delays associated with batch processing.

### AI-Based Threat Detection

* Utilizes the Isolation Forest machine learning algorithm.
* Detects suspicious and abnormal activities automatically.
* Identifies unknown threats that may not match predefined rules.

### User and Entity Behavior Analytics (UEBA)

* Learns normal behavior patterns for users and IP addresses.
* Detects unusual login attempts and abnormal activity.
* Helps identify insider threats and compromised accounts.

### Risk Scoring Engine

* Calculates a risk score for every security event.
* Considers failed logins, anomaly detection results, and UEBA findings.
* Assigns severity levels such as LOW, HIGH, and CRITICAL.

### SOAR Automation

* Automatically performs response actions based on risk level.
* Supports:

  * Monitor
  * Rate Limit
  * IP Blocking
* Reduces response time and manual intervention.

### Security Dashboard

* Built using Flask, HTML, CSS, and JavaScript.
* Displays real-time alerts and threat information.
* Provides visibility into:

  * Alert severity
  * Risk scores
  * Attack types
  * UEBA status
  * Anomaly detection status
  * Automated response actions

---

## System Architecture

1. Security logs are generated by the Producer.
2. Logs are streamed through Apache Kafka.
3. The Consumer receives and processes incoming events.
4. Machine learning models analyze activity for anomalies.
5. UEBA evaluates behavioral patterns.
6. The Risk Engine calculates threat severity.
7. The SOAR Engine determines the appropriate response.
8. Alerts are stored and displayed on the dashboard.

## Example Detection Workflow

1. A user generates multiple failed login attempts.
2. The event is sent to Kafka.
3. The Consumer receives the event.
4. The AI model detects unusual behavior.
5. UEBA identifies abnormal activity from the source IP.
6. The Risk Engine increases the threat score.
7. The event is classified as HIGH or CRITICAL.
8. The SOAR Engine automatically blocks the IP address.
9. The alert appears on the dashboard instantly.
